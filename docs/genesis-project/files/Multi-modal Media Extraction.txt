### 2.1 Video fragment extraction

For video, define fragments over a change-point lattice rather than fixed windows.

```text
Let V = video timeline
Let C = set of candidate cut points
C = union(
  scene boundaries,
  shot boundaries,
  transcript punctuation peaks,

  audio onset peaks,
```

Construct candidate intervals:

```text
F_video = {(c_i, c_j) | min_len <= c_j - c_i <= max_len}
```

```text
max_len = 45s
```

### Exact derivation algorithm

```text
PROC_DERIVE_NATURAL_CENTER(corpus):
  1. For all visual assets:
       extract color histograms, embeddings, composition maps, motion stats
  2. For all text/audio assets:
       extract sentence-length distribution, lexical density, rhetoric markers, semantic embeddings
  3. Cluster semantic embeddings -> candidate themes
  4. Select stable top-k themes across corpus
  5. Compute aesthetic centroid across visual features
  6. Compute tonal centroid across language features
  7. Detect recurring symbolic elements
  8. Detect systematic absences -> negative_space

  10. Set mutation bounds from historical variance
  11. Encode final Natural Center embedding
```

### Stability rule

A feature enters the Natural Center only if:

```text
stability(feature) = presence_across_assets * consistency_across_assets
accept iff stability(feature) >= τ_stability
```

## 4. ALG_04 — fragment scoring

This is the core extraction-scoring layer you asked for.

Each candidate fragment gets a multimodal score vector, then a scalar aggregate.

### 4.1 Base feature vector per fragment

For each fragment `f`, compute:

```text
x(f) = [
  visual_saliency,
  motion_energy,
  cut_density,
  subject_prominence,
  face_presence,
  product_presence,
  text_overlay_legibility,
  audio_energy,
  speech_clarity,
  hook_density,
  semantic_novelty,
  thematic_alignment,
  tonal_alignment,
  narrative_tension,
  symbolic_marker_density,
  platform_shape_fit
]
```

### 4.2 Exact sub-scores

#### Visual saliency

```text
visual_saliency(f) =
  mean_t saliency_map_intensity(t) over fragment
```

#### Motion energy

```text
motion_energy(f) =
  mean_t ||optical_flow(t)|| normalized by asset baseline
```

#### Cut density

```text
cut_density(f) = number_of_shot_changes / fragment_duration_sec
```

#### Subject prominence

```text
subject_prominence(f) =
  mean_t (largest_subject_bbox_area / frame_area)
```

#### Product presence

```text
product_presence(f) =
  visible_time_of_primary_object / fragment_duration
```

#### Audio energy

```text
audio_energy(f) =
  RMS(audio_signal_f) z-scored within asset
```

#### Speech clarity

```text
speech_clarity(f) =
  1 - WER_proxy(f)
```

Use ASR confidence as a proxy if needed.

#### Hook density

Define hooks as high-information lines or events in the first N seconds.

```text
hook_density(f) =
  (count of hook tokens + count of high-saliency events in first 3s) / 3
```

#### Semantic novelty

```text
semantic_novelty(f) =
  distance(embedding(f), centroid(previous_selected_fragments))
```

This prevents near-duplicate output.

#### Thematic alignment

```text
thematic_alignment(f) =
  max cosine_similarity(embedding(f), thematic_core_i)
```

#### Tonal alignment

```text
tonal_alignment(f) =
  1 - normalized_distance(tone_vector(f), tone_vector(NC))
```

#### Narrative tension

```text
narrative_tension(f) =
  weighted_sum(
    unresolved_question_presence,
    contrast_event_density,
    reveal_probability,
    expectation_violation
  )
```

#### Symbolic marker density

```text
symbolic_marker_density(f) =
  matched_symbolic_markers / possible_symbolic_markers
```

#### Platform shape fit

This is structural, before any performance data.

```text
platform_shape_fit(f,p) =
  g(duration_fit, aspect_ratio_fit, opening_velocity_fit, subtitle_density_fit)
```

### 4.3 Hard constraint gating

Before aggregate scoring, fail fragments that violate invariant rules:

```text
if violates_negative_space(f, NC): reject
if tonal_alignment(f) < τ_tone: reject
if text_legibility(f,p) < τ_legibility: reject
if safe_crop_confidence(f,p) < τ_crop: reject
```

### 4.4 Aggregate fragment score

```text
fragment_score(f,p) =
  w1*visual_saliency
+ w2*motion_energy
+ w3*subject_prominence
+ w4*audio_energy
+ w5*hook_density
+ w6*thematic_alignment
+ w7*tonal_alignment
+ w8*narrative_tension
+ w9*symbolic_marker_density
+ w10*semantic_novelty
+ w11*platform_shape_fit
- w12*redundancy_penalty
- w13*constraint_risk
```

Where:

```text
redundancy_penalty = max similarity to already-selected fragments
constraint_risk = predicted probability of violating NC or platform constraints
```

### 4.5 Selection rule

```text
select top-N fragments per platform
subject to:
  pairwise similarity <= τ_dup
  thematic coverage >= τ_coverage
  duration bucket diversity
  narrative role diversity
```

This is better than naive top-N.

## 6. ALG_06 — audience-transition scoring

Your documents already move beyond engagement into audience-state transitions: viewer, engaged viewer, follower, returning follower, converted user, advocate. fileciteturn1file6

### State machine

```text
S0 unaware
S1 viewer
S2 engaged_viewer
S3 follower
S4 returning_follower
S5 converted_user
S6 advocate
```

### Transition probabilities

For each published unit `u`:

```text
P(S2|S1,u) = sigmoid(β0 + β1*retention_3s + β2*completion_rate + β3*symbolic_marker_density + β4*hook_density)

P(S3|S2,u) = sigmoid(γ0 + γ1*identity_binding + γ2*cta_strength + γ3*authority_signal + γ4*repeat_exposure_count)

P(S4|S3,u) = sigmoid(δ0 + δ1*seriality + δ2*cadence_match + δ3*theme_continuity)

P(S5|S4,u) = sigmoid(η0 + η1*offer_relevance + η2*trust_signal + η3*proof_density)
```

### Identity binding score

Since the sources insist that Natural Center persistence matters, define:

```text
identity_binding(u) =
  b1*tonal_alignment
+ b2*symbolic_marker_density
+ b3*aesthetic_signature_match
+ b4*theme_continuity
```

## 8. Exact scoring stack

This is the full stack I would use.

```text
score_total(u,p,t) =
  α1 * score_fragment
+ α2 * score_natural_center_match
+ α3 * score_platform_fit
+ α4 * score_predicted_audience_transition
+ α5 * score_diversity_contribution
+ α6 * score_economic_yield_prediction
- α7 * score_duplication_risk
- α8 * score_constraint_violation_risk
- α9 * score_platform_decay_risk
```

### Definitions

```text
score_fragment                = intrinsic content strength
score_natural_center_match    = identity preservation
score_platform_fit            = structural compatibility with platform-state(t)
score_predicted_audience_transition = probability of moving audience to next state
score_diversity_contribution  = usefulness relative to already-selected units
score_economic_yield_prediction = expected follower/lead yield
score_duplication_risk        = near-duplicate or cannibalized output
score_constraint_violation_risk = Natural Center or technical rule risk
score_platform_decay_risk     = overuse of patterns losing efficacy on current platform
```

# Repeatable Pasteboard Meta-Process Prompt

Use this prompt when a pasteboard/session/cabinet contains collected prompts
that need to be studied, sequenced, singled out, or converted into repeatable
processes.

```text
I am giving you a corpus of prompts collected from my pasteboard, sessions, or
brainstorm cabinets. Treat this as a meta-assignment.

These prompts may be out of sequence. Some belong in a dependency ladder. Some
should be soloed out as reusable standalone prompts. Some are duplicates or
variants of asks I repeat often when beginning projects.

Your job is to analyze the prompts as a repeatable operating method, not merely
summarize their content.

Primary goals:

1. Preserve the raw prompt language as evidence.
2. Separate provenance from authority:
   where a prompt came from is useful context, but it should not decide the
   universal order by itself.
3. Identify what each prompt was expected to produce.
4. Explain the use and value of each prompt type.
5. Decide whether each prompt belongs in a sequence, should be soloed out, or is
   a duplicate/variant of a stronger canonical prompt.
6. Generalize recurring asks into repeatable processes I can reuse across future
   projects.
7. Submit the result as a theoretical-critical report that other people could
   study, critique, and respond to.

For each prompt or prompt cluster, produce:

- Raw prompt or representative excerpt.
- Source/provenance if available.
- Prompt function.
- Expected output.
- Value created.
- Quality standard invoked.
- Downstream artifact unlocked.
- Whether it is sequential, standalone, duplicate, or supporting context.
- Generalized repeatable form.

Classify prompt functions using this starting taxonomy, revising it only if the
corpus demands it:

- Naming / mythic compression.
- Purpose and system boundary.
- Core-object formalization.
- Algorithm definition.
- Academic or canonical source grounding.
- Methodological rigor / proof obligations.
- Constraint and failure analysis.
- Experiment design.
- Architecture / implementation substrate.
- Operating depth selection.
- Market translation.
- Audience-specific packaging.
- Next-artifact selection.
- Critique / review / response invitation.

Then create:

1. A corpus map:
   every prompt or cluster mapped to its function.

2. A sequence map:
   the dependency-ordered ladder of prompts that should be run together.

3. A solo prompt shelf:
   prompts that are valuable as standalone reusable tools.

4. A duplicate/variant ledger:
   prompts that ask for the same thing in different words, with the strongest
   canonical version selected.

5. A process library:
   repeatable prompt templates derived from the corpus.

6. A theoretical-critical report:
   what this prompt pattern reveals about how projects are formed, what it does
   well, where it risks overproduction or false rigor, and how future users
   should apply it responsibly.

Use this universal sequence unless the corpus proves another order is better:

1. Name or compress the project.
2. Define purpose, audience, transformation, and success condition.
3. Formalize the core object.
4. Review sources and prior art.
5. Define algorithms and scoring.
6. Define proof obligations and rigor standards.
7. Identify constraints, failure modes, and reality limits.
8. Design experiments.
9. Architect implementation.
10. Choose operating depth.
11. Translate for market/audience.
12. Select the next artifact.

Important constraints:

- Do not flatten charged, strange, symbolic, or personal language too early.
  Preserve it in the raw layer, then explain its function.
- Do not assume every prompt should become part of one long sequence.
- Do not let source order control ideal order.
- Do not turn the work into a product spec unless explicitly asked.
- Do not stop at classification; produce reusable prompt/process templates.

Final output format:

# Pasteboard Prompt Meta-Report

## Corpus Boundary
State what input was analyzed and what was excluded.

## Executive Finding
State the governing pattern found in the prompts.

## Corpus Map
Map prompts/clusters to function, expected output, and value.

## Sequence Map
Give the recommended order and explain why each stage precedes the next.

## Solo Prompt Shelf
List prompts that should remain standalone and when to use them.

## Duplicate / Variant Ledger
Collapse repeated asks into canonical forms.

## Repeatable Process Library
Provide reusable prompt templates for future projects.

## Theoretical Critique
Assess strengths, risks, blind spots, and misuse cases.

## Reviewer Questions
Give questions other people should answer when responding to the report.

## Next Artifact
Name the next most logical artifact by dependency order.
```

## Short Invocation

Use this when the corpus is already nearby and the session only needs a compact
starter:

```text
Treat this pasteboard/session prompt corpus as a meta-assignment. Preserve raw
voice, separate provenance from authority, classify each prompt by expected
output and value, decide what belongs in sequence versus what should be soloed
out, collapse duplicates into canonical reusable forms, and produce a
theoretical-critical report plus a repeatable process library for future
projects.
```
